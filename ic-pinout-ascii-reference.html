Overview

The tool generates ASCII package drawings from an ordered list of pin labels. Specify:

  • 2-sided packages (DIP/SOIC style) using the two subcommand
  • 4-sided packages (QFP/QFN style) using the four subcommand
  • Compact layouts with --vertical-spread for reduced width
Always pass one ordered list via --pins, then describe package geometry with --counts. Control traversal direction with --sequence and --start.

Basic Usage

python ic_pinout_ascii.py two --name "IC_NAME" --pins "A,B,C,D,E,F" --counts "3,3" --sequence ccw --start left-top
python ic_pinout_ascii.py four --name "IC_NAME" --pins "A,B,C,D,E,F,G,H,I,J,K,L" --counts "3,3,3,3" --sequence ccw --start left-top
python ic_pinout_ascii.py four --vertical-spread --name "IC_NAME" --pins "A,B,C,D,E,F,G,H,I,J,K,L" --counts "3,3,3,3" --sequence cw --start top-left

Numbering Schemas

Pass --numbering <schema> to either subcommand. Default is ccw. This controls how pin numbers are displayed (not how pins are placed from --pins).

ccw (default)

Counterclockwise from pin 1 at top-left. Standard for DIP, SOIC, QFP, QFN packages.

  • 2-sided: Left column 1–n top→bottom, right column (2n)–(n+1) top→bottom.
  • 4-sided: Left 1–nl, bottom nl+1–…, right decreasing, top decreasing.
cw

Clockwise from pin 1 at top-left of the top side. Meaningful for 4-sided only (identical to ccw for 2-sided).

  • 4-sided: Top 1–nt left→right, right nt+1–…, bottom decreasing, left decreasing.
seq

Each side numbered independently from 1. Useful for connectors, headers, and IDC footprints.

none

Pin numbers suppressed. Labels only. Reduces diagram width when numbering adds no value.

Reference: ATtiny Packages

Command: ATtiny412

python ic_pinout_ascii.py two --name "ATtiny412" --pins "VDD,PA6,PA7,PA1,PA2,PA0/RESET/UPDI,PA3/EXTCLK,GND" --counts "4,4" --sequence ccw --start left-top

Command: ATtiny816 Standard

python ic_pinout_ascii.py four --name "ATtiny816" --pins "PA2,PA3/EXTCLK,GND,VDD,PA4,PA5,PA6,PA7,PB5,PB4,PB3/TOSC1,PB2/TOSC2,PB1,PB0,PC0,PC1,PC2,PC3,PA0/RESET/UPDI,PA1" --counts "5,5,5,5" --sequence ccw --start left-top

Command: ATtiny816 Vertical Spread

python ic_pinout_ascii.py four --vertical-spread --name "ATtiny816" --pins "PA2,PA3/EXTCLK,GND,VDD,PA4,PA5,PA6,PA7,PB5,PB4,PB3/TOSC1,PB2/TOSC2,PB1,PB0,PC0,PC1,PC2,PC3,PA0/RESET/UPDI,PA1" --counts "5,5,5,5" --sequence ccw --start left-top

Understanding --sequence and --start

The --sequence option controls how --pins are distributed around the package, and --start specifies the starting endpoint.

ccw (Counterclockwise)

Consume one complete side at a time, traversing counterclockwise.

Valid --start values:

  • 2-sided: left-top, right-bottom
  • 4-sided: left-top, bottom-left, right-bottom, top-right

cw (Clockwise)

Consume one complete side at a time, traversing clockwise.

Valid --start values:

  • 2-sided: right-top, left-bottom
  • 4-sided: top-left, right-top, bottom-right, left-bottom

alternating

Round-robin across sides in traversal order. The --start value is used to infer the traversal direction (ccw or cw), then round-robin is applied.
